# 气泡文本优化说明

## 🎯 优化目标
将气泡文本按标点符号分割成更短的短语，提高CLIP匹配精度，增加气泡数量。

## 🔧 主要改进

### 1. 文本分割功能
- **新增函数**: `splitSentenceIntoPhrases()`
- **分割规则**: 按标点符号 `[，。！？；：、]` 分割
- **长度控制**: 短语长度3-15字符，过长则截断
- **过滤机制**: 自动过滤太短的片段

### 2. CLIP匹配优化
- **降低阈值**: 从0.3降低到0.2，允许更多匹配
- **增加气泡**: 从3-5个增加到6-8个气泡
- **短语匹配**: 每个分割后的短语都进行CLIP匹配

### 3. 后端处理优化
- **文本预处理**: 在后端就进行文本分割
- **更多气泡**: 支持最多8个气泡
- **智能定位**: 结合CLIP和DINOv3的定位算法

## 📊 效果对比

### 优化前
```
原始句子: "构图很好，线条流畅，光影处理得当，细节丰富"
气泡文本: "构图很好，线条流畅，光影处理得当，细节丰富" (1个气泡)
```

### 优化后
```
原始句子: "构图很好，线条流畅，光影处理得当，细节丰富"
分割结果: ["构图很好", "线条流畅", "光影处理得当", "细节丰富"] (4个气泡)
```

## 🚀 性能提升

1. **匹配精度**: 短语匹配比长句匹配更精确
2. **气泡数量**: 从3-5个增加到6-8个
3. **定位准确**: 每个短语都能找到最匹配的图片区域
4. **用户体验**: 更细致的评价标注

## 🔍 使用示例

```typescript
// 输入评价数据
const evaluation = {
  strengths: ["构图很好，线条流畅，光影处理得当"],
  suggestions: ["可以加强质感表现，注意细节刻画"],
  text_region_mapping: [...] // CLIP匹配结果
};

// 输出分割后的气泡文本
const bubbles = filterEvaluationText(evaluation);
// 结果: ["构图很好", "线条流畅", "光影处理得当", "可以加强质感表现", "注意细节刻画"]
```

## ⚙️ 配置选项

- `MIN_CONFIDENCE`: 0.2 (CLIP匹配阈值)
- `maxBubbles`: 8 (最大气泡数量)
- `phraseLength`: 3-15字符 (短语长度范围)
